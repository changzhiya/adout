# Adout 广告拦截 App 设计文档

## 概述

Adout 是一款纯本地运行的 Android 广告拦截应用，目标平台为 MagicOS（荣耀手机）。采用 VPN + DNS 混合方案，无需 Root 权限即可拦截应用启动广告。

## 目标用户

家人朋友，需要简单易用，尽量少的配置步骤。

## 核心功能

### MVP 范围

- 拦截应用启动广告（全屏广告、开屏广告）
- 本地 VPN 方式，无需 Root
- 内置规则 + 本地导入规则文件
- 极简精致的 UI 界面

### 后续计划

- 应用内弹窗广告拦截
- 视频贴片广告拦截
- 浏览器网页广告拦截
- HTTPS MITM 精确拦截

## 技术方案

### 架构：VPN + DNS 混合

```
App 请求 → VPN 拦截 → 解析 DNS → 查询规则 → 放行/拦截
```

**核心组件**：

1. **VPN 服务**：使用 Android `VpnService` 拦截所有网络流量
2. **DNS 解析器**：解析 DNS 请求包，提取域名
3. **规则引擎**：查询内置/导入的规则库，判断是否拦截
4. **UI 界面**：开关控制，状态显示
5. **通知服务**：前台服务，保持存活
6. **证书管理**：高级功能，系统证书安装（可选）

### 技术栈

- **语言**：Kotlin
- **最低 API**：Android 7.0 (API 24)
- **目标 API**：Android 14+ (API 34)
- **核心 API**：`VpnService`
- **规则格式**：Adblock Plus 兼容格式

### 数据流

1. App 发起 DNS 请求（如 `ads.example.com`）
2. VPN 服务拦截流量
3. 解析 DNS 协议，提取域名
4. 查询规则引擎：
   - **命中黑名单** → 返回 `0.0.0.0`（广告被拦截）
   - **未命中** → 转发到真实 DNS（正常放行）

### 规则格式

```bash
# 黑名单规则（拦截）
||ads.example.com^
||tracking.sdk.com^
||ad.doubleclick.net^

# 白名单规则（放行）
@@||important.example.com^

# 通配符匹配
||*.adserver.com^
```

### 规则匹配算法

- **Aho-Corasick 算法**：多模式匹配，一次遍历匹配所有规则
- **域名哈希表**：精确匹配用 O(1) 查找
- **白名单优先**：先检查白名单，命中则直接放行
- **内存缓存**：最近查询结果缓存，避免重复匹配

## UI 设计

### 设计风格

- 极简但精致
- 渐变背景（关闭时紫色系，开启时绿色系）
- 毛玻璃效果按钮
- 一键操作

### 主界面

- 中间大圆按钮：点击开启/关闭 VPN 保护
- 状态显示：保护已开启/关闭
- 统计信息：今日拦截数、规则数量

### 交互逻辑

- 点击中间大圆 → 开启/关闭 VPN 保护
- 首次开启 → 弹出 VPN 权限请求
- 开启成功 → 背景色渐变动画过渡

## HTTPS 处理

### 默认方案：域名黑名单

- 不拦截 HTTPS 内容
- 只按域名整体放行/拦截
- 简单但可能误杀

### 高级方案：系统证书 MITM

- 引导用户安装为系统证书
- 需要 Root 或 ADB 命令
- 配置复杂但拦截彻底

### 实现策略

- 默认使用域名黑名单
- 高级用户可选系统证书
- 两种方案都支持

## 后台运行策略

### MagicOS 兼容

- **前台服务 + 通知栏常驻**：显示一个持久通知，告诉系统"我在工作"
- **引导用户关闭电池优化**：首次启动时引导用户手动设置

### 自动恢复

- VPN 服务意外断开 → 自动重连 + 通知提醒
- 网络切换（WiFi ↔ 移动数据）→ 保持 VPN 连接

## 错误处理

### VPN 服务生命周期

- **启动失败**：VPN 权限被拒绝 → 显示引导界面
- **意外断开**：系统杀死服务 → 自动重连 + 通知提醒
- **其他 VPN 冲突**：检测到其他 VPN → 提示用户关闭
- **网络切换**：WiFi ↔ 移动网络 → 保持 VPN 连接

### DNS 解析异常

- **解析失败**：DNS 包格式错误 → 丢弃，不处理
- **超时处理**：上游 DNS 无响应 → 返回 SERVFAIL
- **IPv6 支持**：AAAA 请求 → 同样规则匹配
- **非 DNS 流量**：其他协议 → 直接转发，不处理

### 规则管理边界

- **规则文件损坏**：解析失败 → 使用内置默认规则
- **规则数量过多**：超过内存限制 → 分页加载
- **规则格式错误**：跳过无效规则，记录日志
- **规则更新**：热加载，无需重启 VPN

## 风险与挑战

### 技术风险

- Android 14 对用户证书限制更严格
- MagicOS 后台运行限制
- VPN 方案本身消耗额外电量（约 5-15%）

### 缓解措施

- 默认使用域名黑名单，避免证书问题
- 前台服务 + 电池优化引导双重保障
- 优化规则匹配算法，减少性能开销

## 成功标准

- 成功拦截国内主流 App 启动广告
- 无明显误杀，App 功能正常
- 后台稳定运行，不被系统杀死
- 用户操作简单，一键开启保护
